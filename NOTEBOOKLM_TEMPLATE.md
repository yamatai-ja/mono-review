# NotebookLM Analysis Template

NotebookLMに投入する分析依頼テンプレートです。

NotebookLMの出力は、本文にそのまま使う事実ではありません。検索意図や論点整理の補助として使い、公開前には必ず人間が公式情報・販売ページ・メーカー情報で確認してください。

## Input Summary

対象キーワード:

関連商品候補:

関連する自サイト記事:

想定読者:

記事タイプ: problem_solution

## Materials To Add

NotebookLMには、可能な範囲で以下を投入します。

- 検索上位記事の本文または要約
- 公式情報
- メーカーFAQ
- 販売ページの確認メモ
- 比較記事
- 関連する自サイト記事
- 読者の不安や質問メモ

注意:

- 著作権保護された本文を丸ごと再利用しない
- NotebookLM出力を本文としてそのまま使わない
- 価格、在庫、保証、対応サービス、仕様は公式確認前に断定しない

## Prompt To NotebookLM

以下の資料をもとに、`problem_solution` 記事を作る前の競合分析をしてください。

本文は書かないでください。SEO記事の構成素材として、論点だけを整理してください。

出力では、確認済み情報と推測を分けてください。公式情報で確認が必要な項目には `要確認` と明記してください。

### 抽出してほしい項目

1. 検索意図の優先順位
2. 読者の不安
3. 上位記事に共通する見出し
4. FAQ候補
5. 比較軸
6. 購入前チェック項目
7. 公式確認が必要な情報
8. 競合記事が弱い論点
9. 自サイトで差別化できる切り口
10. 関連商品を自然に紹介できる文脈
11. 書いてはいけない断定

### 特に重視すること

- 読者が何を不安に思って検索しているか
- 購入前に何を確認しないと失敗しやすいか
- 上位記事が曖昧にしている論点は何か
- FAQとして必要な質問は何か
- 商品を押し売りせず、悩み解決の流れで紹介できるか

## Human Verification After NotebookLM

NotebookLM出力後、人間が確認します。

- 検索意図が実際のSERPとズレていないか
- 公式確認が必要な情報を分けられているか
- 口コミや評判を推測で作っていないか
- 商品仕様を未確認で断定していないか
- 関連商品を主役にしすぎていないか
- FAQが読者不安に対応しているか
- 自サイトの差別化ポイントが明確か

## Notes For GPTs Prompt

NotebookLMの分析結果をGPTsへ渡す場合は、以下を必ず添えます。

- NotebookLMの内容は参考情報であり、事実として断定しない
- 公式確認が必要な情報は本文で断定しない
- 一般論だけで記事を完成させない
- 商品は解決策候補の一つとして扱う
- 内部語や管理メモを本文に出さない
